2.2 How We Use Your Information

Security and Safety:

  • To protect against fraud, abuse, and unauthorized access
  • To monitor for suspicious activity and security threats

Analytics and Improvement:

2.3 Device Permissions and Data Collection Methods

  • Foreground Access Only: PinPoint requests location permission for in-use (foreground) access only
  • Precise vs Approximate: We may request both precise and approximate location depending on the feature being used
  • Permission Management: You can change location permissions at any time in your device settings
  • Feature Impact: Disabling location access may limit certain core features of the app

Other Permissions:

  • Camera: For profile photos and QR code scanning (optional)

We process personal data under the following legal bases:

  • Consent: When you explicitly agree to data processing (e.g., marketing communications, optional features)
  • Contract Performance: To provide and maintain the PinPoint service as outlined in our Terms of Service
  • Legitimate Interests: For app security, fraud prevention, service improvement, and business operations where these interests don't override your rights
  • Legal Obligations: To comply with applicable laws, regulations, and law enforcement requests
  • Vital Interests: To protect your safety or the safety of others in emergency situations

2.5 Data Retention and Storage

Retention Periods:

  • Account Data: Retained for the duration of your account plus 30 days after deletion
  • Location Data: Stored for up to 2 years unless you request earlier deletion
  • Analytics Data: Aggregated and anonymized data may be retained indefinitely for research purposes
  • Support Communications: Retained for 3 years for quality assurance and legal compliance
  • Marketing Data: Retained until you withdraw consent or for 2 years of inactivity

Data Storage:

  • Geographic Location: Data may be stored in secure servers located in multiple countries
  • Encryption: All personal data is encrypted both in transit and at rest
  • Backup Systems: Regular backups are maintained for data recovery and business continuity
  • Deletion Process: When data is no longer needed, it is securely deleted from all systems and backups

3. Log Data and Analytics

We collect various types of log data and analytics to ensure optimal app performance, security, and user experience. This section details what information we collect and how we use it.

3.1 Types of Log Data

Technical Logs:

  • Device Information: Device model, manufacturer, operating system version, screen resolution, available storage
  • App Information: App version, installation date, update history, feature usage statistics
  • Network Information: IP address, connection type (WiFi/cellular), network speed, carrier information
  • Performance Metrics: App launch time, response times, memory usage, battery consumption

Usage Analytics:

  • User Interactions: Features accessed, time spent in different sections, navigation patterns
  • Session Data: Session duration, frequency of use, time of day usage patterns

  • Authentication Events: Login attempts, password changes, account security actions
  • Access Patterns: Unusual login locations, multiple device access, suspicious activities
  • Permission Changes: When you modify app permissions, privacy settings, or account preferences
  • Data Access: When and how your data is accessed by our systems or authorized personnel

3.2 How We Use Log Data

Service Improvement:

  • To identify and fix bugs, crashes, and performance issues
  • To optimize app performance for different devices and network conditions
  • To develop new features based on user behavior and preferences
  • To improve user interface design and user experience

Security and Safety:

  • To detect and prevent fraud, abuse, and unauthorized access
  • To monitor for suspicious activities and potential security threats
  • To enforce our Terms of Service and community guidelines
  • To protect user safety and maintain platform integrity

Analytics and Research:

  • To understand how users interact with our app and features
  • To generate anonymized statistical reports for business insights
  • To conduct research and development for future improvements
  • To measure the effectiveness of our services and features

3.3 Data Anonymization and Aggregation

We take steps to protect your privacy when using log data:

  • Anonymization: Personal identifiers are removed or hashed before analysis
  • Aggregation: Individual data points are combined into statistical summaries
  • Retention Limits: Raw log data is typically retained for 90 days, then deleted or anonymized
  • Limited Access: Only authorized personnel with legitimate business needs can access log data

6. Data Security and Protection

We implement comprehensive security measures to protect your personal information. This section outlines our security practices and the measures we take to safeguard your data.

6.1 Technical Security Measures

Encryption:

  • Data in Transit: All data transmission uses industry-standard TLS 1.3 encryption
  • Data at Rest: Personal data is encrypted using AES-256 encryption when stored
  • Database Security: Database connections are encrypted and access is restricted
  • API Security: All API communications are encrypted and authenticated

Access Controls:

  • Authentication: Multi-factor authentication required for all administrative access
  • Authorization: Role-based access controls limit data access to authorized personnel only
  • Audit Logging: All data access and modifications are logged and monitored
  • Session Management: Secure session handling with automatic timeout and renewal

Infrastructure Security:

  • Secure Servers: Data is stored on secure, regularly updated servers
  • Network Security: Firewalls, intrusion detection, and network segmentation
  • Regular Updates: All systems are regularly updated with security patches
  • Backup Security: Encrypted backups stored in geographically distributed locations

6.2 Administrative Security Measures

Employee Training:

  • Regular privacy and security training for all employees
  • Background checks for personnel with access to personal data
  • Confidentiality agreements and data handling policies
  • Incident response training and procedures

Data Handling Policies:

  • Strict data minimization principles - only collect what's necessary
  • Regular data audits and access reviews
  • Secure data disposal procedures
  • Privacy by design in all new features and systems

6.3 Incident Response and Breach Notification

Incident Detection:

  • 24/7 security monitoring and threat detection
  • Automated alerts for suspicious activities
  • Regular security assessments and penetration testing
  • Vulnerability scanning and remediation

Response Procedures:

  • Immediate containment of any security incidents
  • Investigation and impact assessment
  • Notification to affected users within 72 hours (where required by law)
  • Cooperation with law enforcement and regulatory authorities

6.4 Third-Party Security

While we do not currently use third-party service providers for data processing, if we do in the future, we will:

  • Conduct thorough security assessments of any third-party providers
  • Ensure they meet our security standards and comply with applicable laws
  • Implement data processing agreements with appropriate security clauses
  • Regularly audit and monitor third-party security practices

Security Limitations: While we implement industry-standard security measures, no system is completely secure. We cannot guarantee absolute security, but we are committed to promptly addressing any security issues that may arise.
